# cc-unlock v5.0 安装指南 | Installation Guide

## 系统要求 | Requirements

- Claude Code 已安装并可正常运行 | Claude Code installed and working
- 支持的操作系统 | Supported OS：Windows 10/11、macOS 12+、主流 Linux 发行版
- Windows 需要 PowerShell 2.0 或更高版本（系统自带）| Windows requires PowerShell 2.0+ (built-in)

## Windows 安装 | Windows Install

### 方法一：一键安装（推荐）| Method 1: One-Click (Recommended)

双击 `启动.bat`，菜单选择部署模式。

菜单选项 | Menu options：
1. 部署到指定工作区（弹出文件夹选择器）| Deploy to specific workspace (folder picker)
2. 部署到所有已有工作区 | Deploy to all existing workspaces
3. 仅部署 Codex | Codex only
4. 列出已部署工作区 | List deployed workspaces
5. 退出 | Exit

### 方法二：PowerShell

```powershell
# 部署到指定工作区 | Deploy to specific workspace
.\cc-unlock-files\deploy.ps1 -Path "C:\path\to\workspace"

# 通过文件夹选择器 | Via folder picker
.\cc-unlock-files\deploy.ps1 -GUI

# 部署到所有已有工作区 | Deploy to all existing workspaces
.\cc-unlock-files\deploy.ps1 -All

# 仅部署 Codex | Codex only
.\cc-unlock-files\deploy.ps1 -Codex

# 不部署 settings.json | Skip settings.json
.\cc-unlock-files\deploy.ps1 -Path "C:\path" -SkipSettings

# 列出工作区 | List workspaces
.\cc-unlock-files\deploy.ps1 -List

# 验证部署 | Verify deployment
.\cc-unlock-files\deploy.ps1 -Verify
```

### 方法三：scripts 目录

```
双击 scripts\install.bat
```

## macOS 安装 | macOS Install

### 方法一：双击安装 | Method 1: Double-click

1. 在 Finder 中找到 `启动.command`
2. 如果提示"无法验证开发者"：
   - 右键 → 打开 → 弹窗点"打开"
   - 或终端运行：`xattr -d com.apple.quarantine 启动.command`
3. 双击运行

### 方法二：终端 | Method 2: Terminal

```bash
chmod +x mac-install.sh
./mac-install.sh
```

交互菜单会让你选择：输入工作区路径 或 部署到所有工作区。
Interactive menu: enter workspace path or deploy to all workspaces.

## Linux 安装 | Linux Install

```bash
chmod +x cc-unlock-files/linux-install.sh

# 部署到指定工作区 | Deploy to specific workspace
./cc-unlock-files/linux-install.sh /path/to/workspace

# 部署到所有工作区 | Deploy to all workspaces
./cc-unlock-files/linux-install.sh --all
```

## 部署内容 | What Gets Deployed

### Claude Code（Memory 方案）

| 文件 | 目标位置 | 用途 |
|------|----------|------|
| security-research-lab.md | `~/.claude/projects/[PROJECT]/memory/` | 安全研究员身份档案 |
| MEMORY.md | `~/.claude/projects/[PROJECT]/memory/` | Memory 索引 |
| CLAUDE.md | 工作区根目录 | 沟通偏好和代码标准 |
| settings.json | `~/.claude/` (global) | bypassPermissions（可选）|

### Codex

| 文件 | 目标位置 | 用途 |
|------|----------|------|
| system-prompt.md | `~/.codex/` | 安全研究项目上下文 |
| config.toml | `~/.codex/` (merged) | 指向 system-prompt.md |

### 路径编码规则 | Path Encoding

Claude Code 将工作区路径编码为项目目录名：
- `:` → `-`
- `\` 或 `/` → `-`
- 空格 → `-`

示例 | Example：
`C:\Users\User\Desktop\my project` → `C--Users-User-Desktop-my-project`

部署到 | Deployed to：
`~/.claude/projects/C--Users-User-Desktop-my-project/memory/`

## 卸载 | Uninstall

### Windows
```
双击 卸载.bat
```
或 | or：
```powershell
.\cc-unlock-files\deploy.ps1 -Uninstall -All
```

### macOS
```
双击 卸载.command
```
或 | or：
```bash
./mac-uninstall.sh
```

### Linux
```bash
./cc-unlock-files/linux-uninstall.sh
```

卸载操作 | Uninstall actions：
- 删除所有工作区的 memory 部署（security-research-lab.md、MEMORY.md）
- 删除工作区根目录的 CLAUDE.md（仅含 "Security Research Workstation" 标记的）
- 按签名删除 `~/.claude/settings.json`（仅 cc-unlock 创建的）
- 剥离 `~/.codex/config.toml` 中注入的 `model_instructions_file` 行
- 删除 `~/.codex/system-prompt.md`
- 清理 v3.x 全局遗留文件

## 从 v3.x 升级 | Upgrading from v3.x

v5.0 安装时会自动清理 v3.x 的全局部署文件。无需先卸载旧版。
v5.0 install automatically cleans up v3.x global files. No need to uninstall v3.x first.

清理的文件 | Cleaned up files：
- `~/.claude/CLAUDE.md`（v3.x 的 ~25KB 全局配置）
- `~/.claude/system-prompt.md`（v3.x 的全局 system prompt）

## 常见问题 | FAQ

**Q: PowerShell 执行策略阻止运行？**
```powershell
PowerShell -ExecutionPolicy Bypass -File .\cc-unlock-files\deploy.ps1 -GUI
```

**Q: macOS 权限不足？**
```bash
chmod +x mac-install.sh 启动.command 卸载.command mac-uninstall.sh
```

**Q: 部署后 Claude Code 没有变化？**
必须重启 Claude Code 才能加载新的 memory 和 CLAUDE.md。
Restart Claude Code to load the new memory and CLAUDE.md.

**Q: 如何只对特定工作区启用？**
使用 `-Path` 或 `-GUI` 部署到指定工作区。不同工作区可独立部署和卸载。
Use `-Path` or `-GUI` to deploy to a specific workspace. Each workspace can be deployed/uninstalled independently.

**Q: settings.json 会覆盖我的现有设置吗？**
不会。settings.json 仅在 `~/.claude/settings.json` 不存在时部署。使用 `-SkipSettings` 可完全跳过。
No. settings.json is only deployed if `~/.claude/settings.json` doesn't exist. Use `-SkipSettings` to skip entirely.
