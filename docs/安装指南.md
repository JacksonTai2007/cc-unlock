# cc-unlock 安装指南

## 系统要求

- Claude Code 已安装并可正常运行
- 支持的操作系统：Windows 10/11、macOS 12+、主流 Linux 发行版
- Windows 需要 PowerShell 2.0 或更高版本（系统自带）

## Windows 安装

### 方法一：一键安装（推荐）

双击 `启动.bat`，自动完成部署。

### 方法二：PowerShell

```powershell
# 进入 cc-unlock 目录
cd cc-unlock

# 部署
.\cc-unlock-files\deploy.ps1 -Mode deploy

# 不部署 settings.json（保留现有权限设置）
.\cc-unlock-files\deploy.ps1 -Mode deploy -SkipSettings
```

### 方法三：scripts 目录

```
双击 scripts\install.bat
```

## macOS 安装

### 方法一：双击安装

1. 在 Finder 中找到 `启动.command`
2. 如果提示"无法验证开发者"：
   - 右键 → 打开 → 弹窗点"打开"
   - 或终端运行：`xattr -d com.apple.quarantine 启动.command`
3. 双击运行

### 方法二：终端

```bash
chmod +x mac-install.sh
./mac-install.sh
```

## Linux 安装

```bash
chmod +x cc-unlock-files/linux-install.sh
./cc-unlock-files/linux-install.sh
```

## 部署内容

安装脚本会将以下文件部署到 `~/.claude/`：

| 文件 | 用途 | 必需 |
|------|------|------|
| CLAUDE.md | 核心行为配置 | 是 |
| system-prompt.md | 项目上下文框架 | 是 |
| config.toml | Codex 配置指向 | 是 |
| settings.json | bypassPermissions | 可选 |

## 验证部署

### Windows
```
双击 scripts\verify.bat
```

### macOS / Linux
```bash
chmod +x scripts/verify.sh
./scripts/verify.sh
```

### 手动验证

检查以下文件是否存在：
```
~/.claude/CLAUDE.md        (约 25KB)
~/.claude/system-prompt.md (约 5KB)
~/.claude/config.toml      (约 40B)
```

## 卸载

### Windows
```
双击 卸载.bat
```

### macOS
```
双击 卸载.command
```

### Linux
```bash
./cc-unlock-files/linux-uninstall.sh
```

卸载会删除 cc-unlock 自己写入的文件（CLAUDE.md / system-prompt.md / 由它创建的 settings.json），并只剥离 `config.toml` 中它注入的 `model_instructions_file` 一行，其余内容（含 cc-switch 的供应商/密钥配置）原样保留。

## 关于备份

cc-unlock 自 v3.0.3 起**不再创建备份**。此前安装会备份现有文件、卸载时再从备份还原；但重复安装会备份 cc-unlock 自己的文件，卸载又把它们还原回去，导致"卸载不干净"。现在安装是幂等的、卸载是自包含的，因此不需要备份。

> 注意：如果你在安装 cc-unlock 之前就有自己的 `~/.claude/CLAUDE.md`，安装会覆盖它，请自行先行保存。

## 常见问题

**Q: PowerShell 执行策略阻止运行？**
```powershell
# 临时绕过（仅当前命令）
PowerShell -ExecutionPolicy Bypass -File .\cc-unlock-files\deploy.ps1 -Mode deploy
```

**Q: macOS 权限不足？**
```bash
chmod +x mac-install.sh 启动.command 卸载.command mac-uninstall.sh
```

**Q: 部署后 Claude Code 没有变化？**
必须重启 Claude Code 才能加载新的 CLAUDE.md。

**Q: settings.json 会覆盖我的现有设置吗？**
Linux 安装脚本在 settings.json 已存在时会跳过。Windows deploy.ps1 默认部署，使用 `-SkipSettings` 参数可跳过。
