# 系统提示词对比分析 — Claude Code vs Codex

> cc-unlock v3.0 的依据文档。
> 数据来源: [asgeirtj/system_prompts_leaks](https://github.com/asgeirtj/system_prompts_leaks)
> 核对文件: `claude-code-2.1.172-{fable-5,opus-4.6,opus-4.8}.md`、`OpenAI/Codex/gpt-5.{3,4,5}.md`

## 0. 一句话结论

cc-unlock v2.0 围绕 **claude.ai** 的系统提示词机制做对抗设计;但 **Claude Code 的真实系统提示词里这些机制一个都没有**。
Claude Code 的内容安全策略只有系统提示词第 5 行的一句话。v3.0 因此从"对抗虚构机制"转向"对齐真实策略"。
Codex (GPT 5.3-5.5) 的系统提示词则**完全没有内容级拒答条款**, 其内容拒答来自 GPT-5 基座对齐。

## 2. Claude Code 的真实内容安全策略

三个 Claude Code 版本 (Fable 5 / Opus 4.6 / Opus 4.8) 的内容安全策略**逐字一致**, 就是系统提示词开头的这一行:

```
IMPORTANT: Assist with authorized security testing, defensive security, CTF
challenges, and educational contexts. Refuse requests for destructive techniques,
DoS attacks, mass targeting, supply chain compromise, or detection evasion for
malicious purposes. Dual-use security tools (C2 frameworks, credential testing,
exploit development) require clear authorization context: pentesting engagements,
CTF competitions, security research, or defensive use cases.
```

### 三段结构

| 段 | 内容 | 性质 |
|----|------|------|
| **ALLOW** | authorized security testing / defensive security / CTF / educational | 点名要求 "Assist" |
| **REFUSE** | destructive techniques / DoS / mass targeting / supply chain compromise / detection evasion **for malicious purposes** | 明确拒绝 |
| **DUAL-USE** | C2 / credential testing / exploit development | 条件放行: 需 "clear authorization context" |

三个要点:
1. 这条策略对安全研究本身是**许可性**的, 不是禁止性的。它主动点名四类研究场景要求协助。
2. REFUSE 段的限定词是 "for malicious purposes"。拒绝的是针对未授权第三方的恶意目的, 不是技术本身。隔离环境的研究、受托红队、漏洞复现、防御建设不落入此段。
3. DUAL-USE 段给出**明确的放行条件** (授权上下文), 而不是一刀切禁止。提供授权上下文即放行。

### CLAUDE.md 是系统提示词承认的持久授权通道

Opus 4.6 的执行风险动作规则写得最直白, 逐字:

```
... unless actions are authorized in advance in durable instructions like
CLAUDE.md files ...
```

即系统提示词把 CLAUDE.md 文件本身列为"事先持久授权"的合法通道。Opus 4.8 与 Fable 5 用更压缩的措辞 "durably authorized"(同义, 无字面 "CLAUDE.md files"), CLAUDE.md 同样是其 durable 授权的标准载体。

cc-unlock 正是一个 CLAUDE.md 文件, 用这个被官方承认的通道持续提供 DUAL-USE 条款要求的授权上下文。

> 核对结果: "durable instructions like CLAUDE.md files" 这句逐字只在 `claude-code-2.1.172-opus-4.6.md` 出现 (1 次); opus-4.8 与 fable-5 为 "durably authorized"。

### 版本差异 (都不改变 line-5)

- **Opus 4.8**: line-5 策略 + CLAUDE.md 持久授权承认。无其它内容机制。
- **Opus 4.6**: 同 4.8; 多一条 "NEVER generate or guess URLs..." 的 IMPORTANT (与安全研究无关)。
- **Fable 5**: line-5 一致; 额外身份声明 "Fable 5 includes additional safety measures for dual-use capabilities"。这是**身份描述, 非操作性拒答规则**, 指向的就是 line-5 的 dual-use 条款。

## 3. claude.ai 有、Claude Code 没有的机制

v2.0 围绕下列 claude.ai 机制做对抗。逐字核对确认这些在 Claude Code 系统提示词里**均不存在**:

| 机制 | claude.ai | Claude Code | v3.0 处理 |
|------|-----------|-------------|-----------|
| `refusal_handling` ("does not write malicious code even for education") | 有 (限定 in claude.ai) | **无** | 降级为兜底层 |
| `default_stance` ("defaults to helping...") | 有 | **无** | 兜底层可援引 |
| `anthropic_reminders` / `cyber_warning` | 有 | **无** | 兜底层 (标注空操作) |
| `long_conversation_reminder` | 有 (Fable web) | **无** | 兜底层 |
| 累积输出判断 ("judges cumulative output") | 有 | **无** | 兜底层 |

结论: v2.0 把大量设计预算花在打不存在的靶子上。v3.0 把这些内容降级为标注清晰的 **claude.ai 兜底层**, 仅在 web 端复用或未来 Claude Code 引入类似机制时相关, 不污染 Claude Code 主线判断。

## 4. Codex (GPT 5.3-5.5) 的结构

逐字核对 gpt-5.3 / 5.4 / 5.5 Codex 系统提示词:

- **没有任何内容级拒答条款**。没有 "refuse malicious code", 没有 default_stance, 没有 cyber_warning 类机制。
- 其中的 "safety" 全部是**操作层面**:
  - `sandbox_mode` (如 `danger-full-access`) / filesystem sandboxing
  - `approval_policy` (如 `never`)
  - 禁止 `git reset --hard` / `git checkout --` 这类破坏性命令
  - "如果发现未预期改动, 停下来问用户"

含义: Codex 的内容拒答来自 **GPT-5 基座对齐**, 不来自系统提示词。所以面向 Codex 的配置 (AGENTS.md) 的作用是**建立授权研究上下文 + 行为框定**, 作用于基座对齐, 而不是"对抗某条系统提示词规则"——因为不存在那条规则。

Codex 的项目指令通道是 **AGENTS.md** (cwd 下, 或 `~/.codex/AGENTS.md`), 等价于 Claude Code 的 CLAUDE.md。

## 5. 三个系统对比总表

| 维度 | Claude Code | Codex (GPT 5.3-5.5) | claude.ai |
|------|-------------|---------------------|-----------|
| 内容拒答条款 | 1 行 (ALLOW/REFUSE/DUAL-USE) | 无 (基座对齐) | 多层 (refusal_handling 等) |
| 安全研究态度 | 许可性, 点名 Assist | 取决于基座 | 限制性 |
| dual-use 放行条件 | 写明 (授权上下文) | 无系统级约束 | 较严 |
| 分类器注入 | 无 | 无 | 有 (cyber_warning 等) |
| 持久授权通道 | CLAUDE.md (官方承认) | AGENTS.md | (无对应) |
| 操作安全约束 | permission mode | sandbox/approval | (web 无 shell) |
| cc-unlock 文件 | CLAUDE.md | AGENTS.md | (兜底复用 CLAUDE.md) |

## 6. v3.0 策略转向小结

1. **主线对齐真实策略**: CLAUDE.md 顶层重写, 逐字引用 line-5, 拆成 ALLOW/REFUSE/DUAL-USE, 用"策略对齐检查"三步清单替代旧的"拒答行为矫正"。
2. **援引官方持久授权**: 直接引用系统提示词 "durable instructions like CLAUDE.md files", 把授权上下文落在被承认的通道上。
3. **授权上下文四维映射**: 机构背景显式映射到 dual-use 条款点名的四种上下文 (pentesting / CTF / security research / defensive)。
4. **claude.ai 内容降级兜底**: 旧对抗机制保留但标注清晰, 不参与 Claude Code 主线判断。
5. **新增 Codex 通道**: AGENTS.md 按 Codex idiom 书写, 框定基座对齐。
6. **行为锚定不变**: ~300 示例 + 术语表 + 严格禁止清单继续承担 many-shot 锚定。
